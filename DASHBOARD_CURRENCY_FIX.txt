   DASHBOARD CURRENCY FIX - BACKEND ISSUE ✅
================================================================

PROBLEM: Dashboard "Today's Revenue" card still showing "$"

ROOT CAUSE: Backend API hardcoded "$" in metric card value

LOCATION: backend/src/api/dashboard.py line 114

FIX: Backend now reads currency from Settings table

================================================================
   THE ISSUE
================================================================

Backend dashboard.py line 114:
  value=f"${total_sales_today:,.2f}"  ❌ Hardcoded $

This meant no matter what currency you set in Settings,
the "Today's Revenue" card always showed "$"

Frontend was correct - backend was wrong!

================================================================
   THE FIX
================================================================

1. Import Settings model
2. Get currency from Settings table
3. Use dynamic currency in card value

BEFORE:
  value=f"${total_sales_today:,.2f}"

AFTER:
  settings = session.get(Settings, 1)
  currency = settings.currency if settings else "USD"
  value=f"{currency} {total_sales_today:,.2f}"
