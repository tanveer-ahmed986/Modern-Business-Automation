================================================================================
  WHAT'S NEW IN THIS VERSION
================================================================================

[CRITICAL FIX] Login Hang Issue Resolved
  - Fixed the issue where system hangs after password change
  - Implemented automatic recovery from unresponsive backend
  - No more manual restarts needed!

[NEW] Auto-Recovery Watchdog Service
  - Monitors backend health every 30 seconds
  - Automatically restarts if backend becomes unresponsive
  - Logs all recovery events to watchdog.log
  - Use AUTO_START_WITH_RECOVERY.bat for best experience

[ENHANCED] Smart Login with Timeout Protection
  - 30-second timeout prevents infinite waiting
  - Automatic retry with smart backoff
  - Clear error messages with retry button
  - Server health check before login

[IMPROVED] Backend Connection Management
  - Connection pooling prevents resource exhaustion
  - Automatic cleanup of stale connections
  - Handles 30+ concurrent users reliably
  - Better memory management

[ADDED] Enhanced Health Monitoring
  - /health endpoint for system monitoring
  - Database connectivity verification
  - Fast 5-second health checks
  - Used by watchdog for auto-recovery

================================================================================
  QUICK START GUIDE
================================================================================

For New Installations:
  1. Extract MBAS_v1.0.8_Production_Ready.zip
  2. Read INSTALLATION_GUIDE.txt
  3. Run INSTALL_BACKEND_FIXED.bat
  4. Run AUTO_START_WITH_RECOVERY.bat (Recommended!)
  5. Open http://127.0.0.1:5173
  6. Login with: admin / admin123

For Existing Users Upgrading:
  1. Stop current MBAS (STOP_MBAS.bat)
  2. Backup your mbas_database.db file
  3. Extract new version files (overwrite)
  4. Run AUTO_START_WITH_RECOVERY.bat
  5. Your data is preserved automatically

================================================================================
  NEW STARTUP OPTIONS
================================================================================

Option 1: AUTO_START_WITH_RECOVERY.bat (RECOMMENDED)
  - Starts backend with watchdog protection
  - Starts frontend automatically
  - Auto-recovers from hangs
  - Best for production use
  - Logs events to watchdog.log

Option 2: START_MBAS.bat (Standard)
  - Traditional manual start
  - No auto-recovery
  - You manage restart manually
  - Works as before

Option 3: START_WITH_WATCHDOG.bat (Backend Only)
  - Watchdog for backend only
  - Start frontend separately
  - Good for development

================================================================================
  FOR SYSTEM ADMINISTRATORS
================================================================================

Monitoring:
  - Check watchdog.log for recovery events
  - Monitor /health endpoint (http://127.0.0.1:8000/health)
  - Watch for "Backend restarted successfully" in logs

Configuration Files:
  - backend/watchdog.py - Watchdog settings
  - frontend/src/services/api.ts - Frontend timeout
  - backend/src/core/db.py - Connection pool settings

Troubleshooting:
  1. If login times out: Check if backend is running
  2. If watchdog keeps restarting: Check watchdog.log
  3. If database locked: Watchdog will auto-recover
  4. For persistent issues: Stop all, delete watchdog.log, restart
