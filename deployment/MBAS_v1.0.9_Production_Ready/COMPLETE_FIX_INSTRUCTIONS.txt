You installed MBAS_Setup_v1.2.1_NoTaskkill.exe successfully, but:
  1. Server fails to start within 15 seconds

ROOT CAUSES IDENTIFIED:
-----------------------
1. CRITICAL BUG in mbas_tray.py (line 321):

3. Virtual environment not being used:
   - Tray app uses system Python instead of venv

4. No error logging:
   - Silent failures with no diagnostic information
   - Can't troubleshoot startup issues

================================================================================
    TECHNICAL DETAILS (For Developers)
================================================================================

Bug in Original mbas_tray.py:
------------------------------
Line 321 in auto_open_browser():
    webbrowser.open(self.server_url)  # ERROR: self.server_url doesn't exist

Should be:
    webbrowser.open(self.backend_url)  # This is defined in __init__

This caused the tray app to crash when trying to auto-open the browser,
which made it appear like the server wasn't starting.

Timeout Issue:
--------------
Original timeout: 15 seconds
Actual startup time (first run): 20-30 seconds

Why so long:
- Database initialization: 5-10 seconds
- License validation: 2-3 seconds
- FastAPI app startup: 3-5 seconds
- Import all modules: 5-10 seconds

Solution: Increase timeout to 30 seconds + better status messages

Virtual Environment Issue:
--------------------------
Original code: self.python_exe = sys.executable
Problem: Uses system Python, not venv Python
Result: Missing packages (pystray, Pillow, psutil)

Fixed code:
if (venv_dir / "Scripts" / "python.exe").exists():
    self.python_exe = str(venv_dir / "Scripts" / "python.exe")

CMD Blinking Issue:
-------------------
Caused by watchdog.py trying to restart a crashed server repeatedly.
Fixed by:
1. Fixing the root cause (tray app crash)
2. Better startup timeout
3. Proper error handling


================================================================================
    FAQ
================================================================================

Q: Why does it take 30 seconds to start?
A: First-time startup needs to:
   - Initialize SQLite database
   - Validate license
   - Import all Python modules (FastAPI, SQLModel, etc.)
   - Start uvicorn server
   This is normal and only happens on first start.

Q: Can I make it start faster?
A: After first start, it should start in 5-10 seconds. The 30-second
   timeout is just a safety margin.

Q: Why doesn't the installer include these fixes?
A: The installer needs to be rebuilt with the fixed files. Use:
   deployment\build_installer\BUILD_FIXED_INSTALLER.bat
   This will create MBAS_Setup_v1.2.2_FIXED.exe

Q: What if I get "Port 8000 in use" error?
A: Another process is using port 8000. Either:
   1. Run STOP_MBAS.bat to kill it
   2. Find and stop the other process using that port
   3. Restart your computer

Q: What if Python imports fail?
A: You might have Python 3.13+ which has compatibility issues.
   MBAS requires Python 3.11 or 3.12. Uninstall Python 3.13 and
   install Python 3.12 from python.org.

Q: Can I run MBAS on startup automatically?
A: Yes, but only AFTER you've verified it starts successfully:
   1. Press Win+R
   2. Type: shell:startup
   3. Create shortcut to START_MBAS_TRAY_FIXED.bat
